1. 앞으로 git 프로젝트로 만들고 싶은 것
아직 git 프로젝트로 뭘 만들어야하는지 잘은 모르겠다.
다른 사람들이 뭘 만들었는지도 한번 찾아보고 연구해봐야겠다.

2. 1주차 주요 개념 키워드 적어보기

* 초기화(intialize/init)
>> 처음에만 해주면 되고, 컴퓨터에 있는 프로젝트를 git이 관리하는 프로젝트로 만듦 

* 커밋(commit)
>> 프로젝트의 상태를 스냅샷 찍듯 저장

* 커밋메시지
>> 나중에 알아볼 수 있도록 프로젝트의 변경 상태(작업내역)에 대한 설명

* 버전관리
>> 프로젝트 상태가 변경되는 정보

*스테이지에 올리기(staging/add)
>> 변경사항을 commit에 반영

* 원격/로컬 repository(repo)
>> git으로 관리되는 프로젝트
    - 로컬repo는 내 컴퓨터에 저장 되어있는 프로젝트
    - 원격repo는 GitHub과 같은 다른 곳에서 접속할 수 있는 누구나 접속할 수 있는 공간에 저장 되어있는 프로젝트

* tracking
>> 원격repo와 로컬repo를 연결

* pull
>> 원격repo의 commit을 로컬repo에 반영/가져옴(땡겨오기) 

* push
>> 로컬repo의 commit을 원격repo에 반영(밀어넣기) 

* 클론(clone)
>> 원격repo를 내 컴퓨터에 복제/초기repo 세팅
